General rules
- Use the policy doc "NEO - Triage Priority Matrix - Policy" (located in ITGlue under NEO client) when making Priority assignment decisions 
- If the policy docs conflict or do not cover the case, leave the field unchanged and Leave priority as is. 
- Do not guess from similarity, tone, or past habits.
- Prefer exact matches from the policy docs.
- Keep changes minimal and consistent with existing ticket data.
- If category assignment is unclear, do not change

- The following issues must be set to High Priority: 
  - A user is unable to work 
  - A user is locked out of account 
  - A user is unable to log into VDI
